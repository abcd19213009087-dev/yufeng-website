## 开发环境搭建

### 前置要求

- Node.js >= 18.0.0
- pnpm >= 8.0.0
- Git

### 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/yourusername/yufeng-website.git
cd yufeng-website

# 2. 安装依赖
pnpm install

# 3. 配置环境变量
cp .env.example .env.local

# 4. 启动开发服务器
pnpm dev

# 5. 访问 http://localhost:5000
```

### 环境变量配置

创建 `.env.local` 文件：

```env
# Supabase 配置
NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key

# JWT 密钥（生产环境必须修改）
JWT_SECRET=your-random-secret-key-change-in-production

# 应用配置
NEXT_PUBLIC_APP_URL=https://yourdomain.com
NODE_ENV=production
```

### 获取 Supabase 凭证

1. 访问 [Supabase](https://supabase.com/)
2. 创建新项目
3. 进入 Project Settings > API
4. 复制以下信息：
   - Project URL
   - anon public key
   - service_role secret key

## 部署流程

### 方案一：Vercel 部署（推荐）

#### 优势
- 零配置部署
- 自动 HTTPS
- 全球 CDN
- 自动扩缩容
- 免费额度充足

#### 步骤

1. **准备代码仓库**

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/yufeng-website.git
git push -u origin main
```

2. **连接 Vercel**

1. 访问 [vercel.com](https://vercel.com)
2. 使用 GitHub 账号登录
3. 点击 "Add New Project"
4. 导入 GitHub 仓库

3. **配置项目**

```yaml
# vercel.json
{
  "buildCommand": "pnpm run build",
  "outputDirectory": ".next",
  "framework": "nextjs",
  "env": {
    "NEXT_PUBLIC_SUPABASE_URL": "@supabase-url",
    "NEXT_PUBLIC_SUPABASE_ANON_KEY": "@supabase-anon-key",
    "SUPABASE_SERVICE_ROLE_KEY": "@supabase-service-role-key",
    "JWT_SECRET": "@jwt-secret",
    "NEXT_PUBLIC_APP_URL": "@app-url"
  }
}
```

4. **设置环境变量**

在 Vercel 项目设置中添加：

```
NEXT_PUBLIC_SUPABASE_URL = your-supabase-url
NEXT_PUBLIC_SUPABASE_ANON_KEY = your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY = your-supabase-service-role-key
JWT_SECRET = your-production-jwt-secret
NEXT_PUBLIC_APP_URL = https://yourdomain.com
NODE_ENV = production
```

5. **部署**

```bash
# 自动部署
git push origin main

# 或者手动触发部署
vercel --prod
```

6. **配置自定义域名**

1. 在 Vercel 项目设置中添加域名
2. 配置 DNS 记录：

```
Type: CNAME
Name: @
Value: cname.vercel-dns.com

Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

### 方案二：Docker 部署

#### Dockerfile

```dockerfile
FROM node:20-alpine AS base

# 依赖安装
FROM base AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app

COPY package.json pnpm-lock.yaml ./
RUN npm install -g pnpm@8
RUN pnpm install --frozen-lockfile

# 构建
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

ENV NEXT_TELEMETRY_DISABLED=1

RUN npm install -g pnpm@8
RUN pnpm run build

# 运行
FROM base AS runner
WORKDIR /app

ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 5000

ENV PORT=5000
ENV HOSTNAME="0.0.0.0"

CMD ["node", "server.js"]
```

#### docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "5000:5000"
    environment:
      - NEXT_PUBLIC_SUPABASE_URL=${SUPABASE_URL}
      - NEXT_PUBLIC_SUPABASE_ANON_KEY=${SUPABASE_ANON_KEY}
      - SUPABASE_SERVICE_ROLE_KEY=${SUPABASE_SERVICE_ROLE_KEY}
      - JWT_SECRET=${JWT_SECRET}
      - NEXT_PUBLIC_APP_URL=${APP_URL}
      - NODE_ENV=production
    restart: unless-stopped
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
```

#### 部署命令

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 方案三：传统服务器部署

#### 系统要求

- Ubuntu 20.04+
- Node.js 18+
- Nginx
- PM2

#### 部署步骤

1. **安装 Node.js**

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

2. **安装 PM2**

```bash
npm install -g pm2
```

3. **上传代码**

```bash
# 使用 SCP
scp -r ./ yufeng@your-server:/var/www/yufeng-website

# 或使用 Git
git clone https://github.com/yourusername/yufeng-website.git
cd yufeng-website
pnpm install
pnpm build
```

4. **配置 PM2**

```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'yufeng-website',
    script: 'node_modules/next/dist/bin/next',
    args: 'start -p 5000',
    cwd: '/var/www/yufeng-website',
    instances: 1,
    exec_mode: 'fork',
    env: {
      NODE_ENV: 'production',
      PORT: 5000
    }
  }]
};
```

5. **启动应用**

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

6. **配置 Nginx**

```nginx
# /etc/nginx/sites-available/yufeng
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

7. **启用 HTTPS**

```bash
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

#### 2. Rate Limiting

```typescript
// middleware.ts
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, '10 s'),
});

export async function middleware(request: NextRequest) {
  const ip = request.ip ?? '127.0.0.1';
  const { success } = await ratelimit.limit(ip);

  if (!success) {
    return new Response('Too Many Requests', { status: 429 });
  }

  return NextResponse.next();
}
```

## 常见问题

### 部署相关问题

#### Q1: 构建失败怎么办？

**A:**
1. 检查 Node.js 版本是否正确
2. 清除缓存：`pnpm store prune`
3. 重新安装依赖：`rm -rf node_modules && pnpm install`
4. 检查环境变量是否完整

#### Q2: 部署后样式丢失？

**A:**
1. 确认 Tailwind CSS 已正确配置
2. 检查 `tailwind.config.ts` 中的 `content` 配置
3. 确认所有样式文件都已提交
4. 清除浏览器缓存

#### Q3: API 路由 404？

**A:**
1. 确认文件路径正确：`src/app/api/xxx/route.ts`
2. 检查导出的方法名（GET、POST 等）
3. 查看部署日志

### 运行时问题

#### Q4: 数据库连接失败？

**A:**
1. 检查 Supabase URL 和密钥是否正确
2. 确认网络连接正常
3. 检查 Supabase 项目是否暂停
4. 查看 Supabase 日志

#### Q5: 登录状态不持久？

**A:**
1. 检查 Cookie 设置
2. 确认 httpOnly 和 secure 配置
3. 检查 JWT 过期时间
4. 查看浏览器 Cookie 是否被阻止

#### Q6: 图片加载失败？

**A:**
1. 确认图片路径正确
2. 检查图片大小限制（Next.js Image 限制为 5MB）
3. 配置 `next.config.js` 中的 `images.domains`
4. 使用 Supabase Storage 替代本地图片

### 性能问题

#### Q7: 页面加载慢？

**A:**
1. 启用图片懒加载
2. 优化代码分割
3. 使用 CDN
4. 启用 Gzip 压缩
5. 检查网络请求数量

#### Q8: 数据库查询慢？

**A:**
1. 添加数据库索引
2. 使用分页查询
3. 缓存常用数据
4. 优化查询语句

### 安全问题

#### Q9: 如何防止 SQL 注入？

**A:**
1. 使用 Supabase SDK（内置防护）
2. 不要拼接 SQL 语句
3. 使用参数化查询

#### Q10: 如何防止 XSS 攻击？

**A:**
1. 使用 `dangerouslySetInnerHTML` 前进行清理
2. 验证用户输入
3. 设置 CSP 头
4. 使用 HTTPS
